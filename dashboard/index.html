Cost Optimization Recommendations
🥇
Batch all anomalies into ONE API call
Currently: 1 Gemini call per manager (5 departments = 5 API calls). Combine all into one structured prompt with per-manager sections.
↓ ~80% API call overhead · same output quality
🥈
Remove redundant flag explanations in prompt
FLAG_EXPLANATIONS are repeated for every transaction in the prompt. Move to a single header definition block — referenced once per flag type.
↓ ~20–30% input token reduction per call
🥉
Switch to structured JSON output mode
Instead of free-text email drafts, use Gemini's JSON response mode to produce structured alert data. Faster inference, lower output tokens, programmatically usable.
↓ ~35% output tokens · faster responses
4
Cache view results to avoid repeat DB queries
On re-runs with unchanged spend.db, skip the SQL view rebuild and DB query — load from a JSON snapshot instead.
↓ DB queries to 0 on unchanged data
5
Shorten timestamp format in prompts
Full ISO 8601 timestamps (26 chars each) are verbose in prompts. Use date + time only: "15 Jun 14:32" saves ~10 chars per transaction row.
↓ ~5–10% input tokens across all calls
